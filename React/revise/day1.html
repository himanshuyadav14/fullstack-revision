Q6 setCount called 3 times — why is result 1 not 3? How to fix? ▾
9/10
✅ What You Got Right
Answer is 1 — React batches state updates
All 3 calls read stale count=0 → all are setCount(1)
Fix: functional updater form prev => prev + 1
⚠️ What Was Missing
Explain WHY functional form works: each call receives the latest QUEUED value, not the stale closure value

// ❌ All read stale count=0 → result: 1

Q7 What triggers a re-render in React? List everything. ▾
6.5/10
⚠️ You Only Listed 2 of 5
✅ All 5 Re-render Triggers
1. State change — setState / useState setter called
2. Props change — parent passes new/different prop value

Q1 What is the Virtual DOM? Walk through the full flow from state change to screen update. ▾
6/10
✅ What You Got Right
Virtual DOM is lightweight JS representation of real DOM

Process is called reconciliation
⚠️ What Was Missing — Full Step-by-Step Flow

2. React creates NEW Virtual DOM tree (re-renders in memory)
3. React DIFFS new vs previous Virtual DOM (reconciliation)
4. Calculates minimum set of changes (patch)
5. ReactDOM applies ONLY those changes to real DOM
6. Browser repaints screen
Diffing Rule 1: Different element types → destroy subtree and rebuild
Diffing Rule 2: Keys identify which list items changed

Q1 Counter — increment, decrement, reset. Count never below 0. ▾
9/10
⚠️ Minor Issues
Use === not == (strict equality always)
Don't import unused hooks (useEffect was imported but unused)
✅ Clean Solution
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)

  const handleDecrement = () => {
    if (count === 0) return  // ✅ use ===
    setCount(prev => prev - 1)
  }

  return (
    <div>
      <p>{count}</p>
      <button onClick={handleDecrement}>Decrement</button>
      <button onClick={() => setCount(prev => prev + 1)}>Increment</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  )
}
Q2 Todo list — add, delete, empty validation, "No todos yet" ▾
9.5/10
⚠️ Minor Improvements
Use functional updater: setTodos(prev => [...prev, newTodo])
onChange={(e) => handleChange(e)} → simplify to onChange={handleChange}
✅ Complete Solution
const ToDo = ({ todo, onDelete }) => (
  <div>
    <span>{todo.name}</span>
    <button onClick={() => onDelete(todo.id)}>Delete</button>
  </div>
)

function App() {
  const [todos, setTodos] = useState([])
  const [input, setInput] = useState('')

  const handleAdd = () => {
    if (!input.trim()) return
    setTodos(prev => [...prev, { id: Date.now(), name: input }]) // functional updater ✅
    setInput('')
  }
  const handleDelete = id => setTodos(prev => prev.filter(t => t.id !== id))

  return (
    <div>
      <input value={input} onChange={handleChange} /> {/* simplified ✅ */}
      <button disabled={!input.trim()} onClick={handleAdd}>Add</button>
      {todos.length === 0
        ? <p>No todos yet</p>
        : todos.map(todo => <ToDo key={todo.id} todo={todo} onDelete={handleDelete} />)
      }
    </div>
  )
}

Q4 Stale closure — setInterval always logs 0. Problem name + fix. ▾
10/10
✅ Perfect Answer — Senior Level
// Problem: stale closure — interval captures count=0 from first render

// Fix 1 — add count to deps (interval restarts every change)
useEffect(() => {
  const id = setInterval(() => console.log(count), 1000)
  return () => clearInterval(id)
}, [count])

// Fix 2 — useRef (better — interval stays stable, always reads fresh value)
const countRef = useRef(count)
