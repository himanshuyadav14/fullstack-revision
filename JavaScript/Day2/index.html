Difference between Microtask and Macrotask Queue?

// Microtask Queue — higher priority
// Promise.then(), async/await, queueMicrotask()

// Macrotask Queue — lower priority
// setTimeout, setInterval, DOM events

Promise.resolve().then(() => console.log("3")); // Microtask
console.log("4"); // Call Stack

Round 2 - Call, Apply, Bind

Q9

What is call()? Show function borrowing!

const person = {
  name: "Himanshu",
  greet: function(city, country) {
    console.log(`Hi I am ${this.name} from ${city}, ${country}`);
  }
}
const person2 = { name: "Rahul" }

// call — executes immediately, args comma separated
person.greet.call(person2, "Jaipur", "India");
// "Hi I am Rahul from Jaipur, India"
Q10

What is apply()? How is it different from call()?

// apply — args in ARRAY
person.greet.apply(person2, ["Jaipur", "India"]);

// call — C for Comma separated
// apply — A for Array

// Real use case:
const nums = [5, 2, 8, 1, 9];
Math.max.apply(null, nums); // 9
Math.max(...nums);          // 9 (modern way)
Q11

What is bind()? How is it different?

// bind — does NOT execute! Returns new function!
const greetRahul = person.greet.bind(person2, "Jaipur", "India");
greetRahul(); // executes later!

// Summary:
// call()  — executes immediately, comma args
// apply() — executes immediately, array args
// bind()  — returns new function, comma args

// Real use case — event listeners
button.addEventListener("click", obj.handleClick.bind(obj));
Q12

Fix this using bind()!

const obj = {
  name: "Himanshu",
  greet: function() { console.log(this.name); }
}
const greet = obj.greet;
greet(); // undefined — fix!
const greet = obj.greet.bind(obj);
greet(); // "Himanshu" ✅
Q13

Use call, apply, bind with calculator!

const calculator = {
  multiply: function(a, b) { return a * b; }
}
const obj1 = { value: 5 };
const obj2 = { value: 10 };
// call
calculator.multiply.call(obj1, obj1.value, obj2.value); // 50

// apply
calculator.multiply.apply(obj1, [obj1.value, obj2.value]); // 50

// bind
const mult = calculator.multiply.bind(obj1, obj1.value, obj2.value);
mult(); // 50

Round 3 - Prototypes & Inheritance

Q14

What is Prototype? What is Prototype Chain?

// Every object has __proto__ pointing to its prototype
// Prototype = shared methods object

const arr = [1, 2, 3];
arr.__proto__ === Array.prototype // true
// map, filter, reduce are on Array.prototype!

// Prototype Chain:
// arr -> Array.prototype -> Object.prototype -> null

// JS looks up the chain if property not found!
Q15

Add greet() to Person using prototype!

function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function() {
  console.log(`Hi I am ${this.name}`);
}

const p1 = new Person("Himanshu", 32);
const p2 = new Person("Rahul", 25);
p1.greet(); // ✅ — shared method!
Q16

Prototypal Inheritance — Student inherits Person!

function Student(name, age, course) {
  Person.call(this, name, age); // borrow constructor
  this.course = course;
}

Student.prototype = Object.create(Person.prototype);
Student.prototype.constructor = Student;

// ES6 way (cleaner):
class Student extends Person {
  constructor(name, age, course) {
    super(name, age);
    this.course = course;
  }
}

const s1 = new Student("Himanshu", 32, "JavaScript");
s1.greet(); // inherited! ✅
Q17

Difference between __proto__ and prototype?

// prototype — on constructor functions
Person.prototype // object where methods are stored

// __proto__ — on instances, links to prototype
const p1 = new Person("Himanshu");
p1.__proto__ === Person.prototype // true!

// prototype = actual object with methods
// __proto__ = link to that object
